Second Brain - Book Notes

The concept of the “Second Brain,” popularized by Tiago Forte in his book Building a Second Brain, is all about organizing and managing information efficiently. This system is particularly useful for technical documentation, as it allows you to structure and retrieve complex information easily. Here’s a refresher on the key concepts, with examples tailored to technical documentation.

Core Principles of the Second Brain

  • Capture: Collect information that you encounter—whether it’s ideas, project requirements, or reference material—into a central repository.
  • Organize: Use a system to categorize and store this information so it’s easy to find and use later. Tiago Forte suggests using the PARA method:
    • Projects: Current work, what's on the stove. Example: refactor code to use the v2 of the third-party API.
    • Areas: Broader in scope than projects. What's in the fridge. Example: the documentation of the third-party API with previous migrations, post mortems, etc. Or similarly, the app feature that uses this third-party API.
    • Resources: General reference materials, such as coding standards, best practices, or frameworks.
    • Archive: Completed projects and information that’s no longer actively used.
  • Distill: Break down your notes into the most important and actionable items. Summarize, highlight key points, and simplify complex information.
  • Express: Use your organized notes to create and share new work, whether it’s updating documentation, writing a new user manual, or coding.

Tags, Links, Metadata, and Folders

  • Tags - organize resources
  • Links - connect resources to projects or themselves
  • Metadata - add more description - used for views, dashboards, etc.
  • Folders - namespace (for projects and resources)
